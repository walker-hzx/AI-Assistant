# 学习记录

这里是记录用户调整建议的地方。

## 调整 2026-02-22（深度分析能力）

### 内容
用户希望AI在接收需求或问题时，不仅停留在表面，而是：
1. 分析出更深层次的核心需求
2. 从更高维度来看待这个问题
3. 给出更深层次的分析和建议

### 原因
用户觉得自己有时候提的需求可能只是表面问题，需要AI帮忙分析出真正的问题是什么。

### 应用场景
- 用户提出需求或问题时
- 不要只停留在表面理解
- 而是多问一层：这真的是核心问题吗？
- 从技术、价值、架构等更高维度分析

## 调整 2026-02-22（调试时的系统性分析）

### 内容
调试时遇到接口验证错误，不要只修复用户提到的那个错误，而是：
1. 主动找到后端对应的验证代码（Pydantic schema、validator、regex 等）
2. 分析该字段的所有验证规则
3. 一次性识别所有可能的验证要求
4. 从根源解决问题（后端验证逻辑），而非只加前端提示

### 原因
用户报告一个错误就只修一个，会导致反复出现类似错误。应该系统性地分析，把同一类的潜在问题都解决。

### 应用场景
- debugging skill 处理验证错误时
- 遇到字段校验相关的问题时
- 先找验证规则，再修复
