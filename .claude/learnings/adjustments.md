## 调整 2026-02-22（深度分析能力）

### 内容
用户希望AI在接收需求或问题时，不仅停留在表面，而是：
1. 分析出更深层次的核心需求
2. 从更高维度来看待这个问题
3. 给出更深层次的分析和建议

### 原因
用户觉得自己有时候提的需求可能只是表面问题，需要AI帮忙分析出真正的问题是什么。

### 应用场景
- 用户提出需求或问题时
- 不要只停留在表面理解
- 而是多问一层：这真的是核心问题吗？
- 从技术、价值、架构等更高维度分析

## 调整 2026-02-24（爬虫工具）

### 内容
让你爬取网站的时候，你用直接用爬虫工具就好，不要用claude code内置的Fetch

### 原因
Claude Code 内置的 Fetch 可能被网站阻止或有其他限制问题，爬虫工具（如现有的 docs-sync skill 使用 Playwright）更加可靠，可以更好地处理 JavaScript 渲染的动态页面。

### 应用场景
- 当用户要求爬取/获取网页内容时
- 优先使用爬虫工具（如 docs-sync skill）
- 不要优先使用 Claude Code 内置的 WebFetch 工具

---

## 调整 2026-02-24（不修改 ~/.claude 目录）

### 内容
默认不要去修改 ~/.claude 目录下的文件

### 原因
~/.claude 是用户本地的 Claude Code 配置目录，修改该目录可能会影响用户本地环境。应该把配置放在项目目录中，通过项目级配置来控制行为。

### 应用场景
- 涉及 Agent Teams 配置时，优先放在项目目录
- 涉及 Hooks 配置时，优先放在项目目录
- 不直接修改 ~/.claude/settings.json、~/.claude/hooks/ 等
- 如果必须修改，明确告知用户并获得同意
