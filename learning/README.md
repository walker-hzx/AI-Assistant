# 学习数据收集模块

> 第一阶段：数据收集机制

## 概述

本模块用于收集用户与 AI 协作过程中的数据，为后续的自适应学习提供素材。

## 数据类型

### 1. 任务数据
- 任务类型（feature/bugfix/refactor/docs/test/research）
- 任务复杂度
- 执行时长
- 执行结果（成功/失败/需调整）

### 2. 偏好数据
- quietMode 使用频率
- 无人值守模式使用情况
- 跳过的环节
- 干预点

### 3. 反馈数据
- 验收标准调整
- 计划修改
- 重新执行次数
- 质量要求

### 4. 交互数据
- 确认类 vs 操作类请求比例
- 用户主动干预的时机
- 用户偏好（简洁/详细）

## 收集时机

| 时机 | 收集内容 |
|------|----------|
| 任务开始 | 任务类型、复杂度、用户指令 |
| 任务阶段切换 | 使用的流程、跳过的环节 |
| 用户干预 | 干预类型、原因、调整内容 |
| 任务完成 | 执行结果、质量反馈、耗时 |

## 数据存储

收集的数据存储在用户主目录：
```
~/.claude/ai-assistant/
├── user-profile.json    # 用户画像
├── task-history/       # 任务历史
└── patterns/          # 学习到的模式
```

## 集成方式

通过 Hook 机制自动收集：
- SessionStart：初始化
- PostToolUse：记录工具使用模式
- Stop：任务完成汇总
