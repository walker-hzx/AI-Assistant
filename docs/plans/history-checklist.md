# 历史问题检查清单

> 从历史执行问题中提取的通用检查项，每次执行前检查

## 使用说明

在 executing-plans 执行每个任务前，检查此清单，避免重复犯同样的错误。

---

## 常见历史问题检查项

### 1. 需求理解类

- [ ] **需求边界不清**
  - 检查：是否明确了输入范围、数据限制？
  - 历史案例：未确认分页数量限制，导致大数据量时性能问题

- [ ] **异常场景遗漏**
  - 检查：是否考虑了错误情况、空值、网络失败？
  - 历史案例：未处理 API 超时，导致页面卡死

- [ ] **功能遗漏**
  - 检查：是否对照需求文档逐一检查？
  - 历史案例：忘记实现批量删除功能

### 2. 代码实现类

- [ ] **边界条件未处理**
  - 检查：空数组、空字符串、0值、null/undefined
  - 历史案例：未处理空数组，导致页面报错

- [ ] **类型错误**
  - 检查：字符串/数字混淆、日期格式、布尔值
  - 历史案例：字符串数字相加导致错误结果

- [ ] **异步处理错误**
  - 检查：async/await、Promise、回调地狱
  - 历史案例：未 await 异步操作，导致数据未加载就渲染

- [ ] **内存泄漏**
  - 检查：定时器未清除、事件监听未移除、闭包引用
  - 历史案例：组件卸载后定时器还在运行

### 3. 接口/API 类

- [ ] **接口错误未处理**
  - 检查：4xx/5xx 错误、超时、网络断开
  - 历史案例：未处理 404 错误，导致白屏

- [ ] **数据格式错误**
  - 检查：字段名拼写、数据类型、嵌套结构
  - 历史案例：字段名大小写不一致，导致数据不显示

- [ ] **缺少 Loading 状态**
  - 检查：异步操作时是否有 loading 提示
  - 历史案例：接口慢时用户以为页面卡死

### 4. UI/UX 类

- [ ] **响应式问题**
  - 检查：移动端适配、不同屏幕尺寸
  - 历史案例：只在桌面端测试，移动端布局错乱

- [ ] **表单验证不完整**
  - 检查：必填项、格式校验、长度限制
  - 历史案例：前端未校验，提交后后端报错

- [ ] **错误提示不友好**
  - 检查：错误信息是否用户可理解
  - 历史案例：直接显示技术错误信息，用户看不懂

### 5. 测试类

- [ ] **测试覆盖不足**
  - 检查：是否只测了正常路径？边界条件呢？
  - 历史案例：测试用例太少，上线后问题频发

- [ ] **测试不真实**
  - 检查：是否 mock 了太多，与真实场景不符？
  - 历史案例：mock 数据格式与真实 API 不一致

- [ ] **未做手动验证**
  - 检查：自动化测试通过后，是否手动跑过？
  - 历史案例：测试通过但实际页面报错

### 6. 性能类

- [ ] **大数据量处理**
  - 检查：列表数据量大时是否分页/虚拟滚动？
  - 历史案例：一次性加载 10万条数据，页面卡死

- [ ] **重复渲染**
  - 检查：是否有不必要的组件重渲染？
  - 历史案例：state 更新导致整个页面重渲染

- [ ] **资源加载**
  - 检查：图片/资源是否过大？是否懒加载？
  - 历史案例：首页加载 10MB 图片，加载极慢

## 执行前检查流程

```
开始执行任务
    ↓
查看历史问题检查清单
    ↓
对照当前任务类型，检查相关项
    ↓
标记需要注意的问题
    ↓
实现时重点注意
    ↓
完成后验证是否避免
```

---

## 更新机制

每次 session-diagnosis 复盘后，如果发现新的常见问题，添加到此清单。
