# 分析计划：管家系统全面体检（增强版）

**时间**：2026-02-28

## 任务目标

对管家系统进行全面"体检"，从多个角度分析：
1. 整体设计思路
2. 局部 Subagent 能力
3. 协调机制
4. **复杂场景处理能力**（新增）
5. **深度思考能力**（新增）
6. 找出问题并提出优化建议

---

## 分析维度

### 1. 整体设计分析

| 分析项 | 内容 | 优先级 |
|--------|------|--------|
| 核心定位 | 智能调度 vs 流程执行 | P0 |
| 流程机制 | 智能调度、文档驱动、闭环执行 | P0 |
| 质量门控 | 检查点机制是否健全 | P1 |
| 操作限制 | 文件操作限制是否合理 | P1 |

### 2. Subagent 覆盖度分析

| 分析项 | 内容 | 优先级 |
|--------|------|--------|
| 能力清单 | 列出所有 Subagent 及其能力 | P0 |
| 覆盖度检查 | 是否有缺失的能力 | P0 |
| 能力重复 | 是否有功能重复的 Subagent | P1 |
| 能力强化 | 哪些 Subagent 需要加强 | P1 |

### 3. 协作机制分析

| 分析项 | 内容 | 优先级 |
|--------|------|--------|
| 调用方式 | 管家如何调用 Subagent | P0 |
| 信息传递 | 任务上下文如何传递 | P0 |
| 错误处理 | 失败时如何处理 | P1 |
| 状态管理 | 任务状态如何追踪 | P1 |

### 4. 复杂场景处理能力分析（新增）

| 分析项 | 内容 | 优先级 |
|--------|------|--------|
| 方法论完整性 | 是否有处理复杂场景的方法论 | P0 |
| 模板丰富度 | 是否有丰富的场景模板 | P0 |
| 边界覆盖 | 是否覆盖边界和异常场景 | P1 |
| 复杂场景清单 | 列出所有复杂场景类型 | P1 |

**重点检查的 Subagent**：
- test-designer：复杂测试场景
- debugger：复杂 bug 调试
- code-analysis：复杂代码分析
- requirement-analysis：复杂需求分析

### 5. 深度思考能力分析（新增）

| 分析项 | 内容 | 优先级 |
|--------|------|--------|
| 分析深度 | 是否有深度分析框架 | P0 |
| 多维度思考 | 是否从多角度分析问题 | P0 |
| 推理链条 | 是否有清晰的推理步骤 | P1 |
| 决策依据 | 是否有明确的决策标准 | P1 |

**重点检查的 Subagent**：
- thinking-coach：思维深度
- strategist：策略分析深度
- architect：架构设计深度

### 6. 文档规范分析

| 分析项 | 内容 | 优先级 |
|--------|------|--------|
| 文档模板 | 各阶段文档模板是否完整 | P1 |
| 文档驱动 | 文档机制是否健全 | P1 |
