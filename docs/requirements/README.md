## 使用说明

### 制定计划时

1. **读取本文档**：`docs/requirements/README.md`
2. **只基于"当前确认需求"制定计划**
3. **忽略"已作废需求"**
4. **如果需求状态不明确，询问用户确认**

### 需求变更时

1. **新增需求**：添加到"当前确认需求"部分
2. **需求完成**：更新状态为 ✅ 已完成
3. **需求作废**：从"当前确认需求"移到"已作废需求"，说明原因
