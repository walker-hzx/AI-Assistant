# 查看需求

查看当前正在进行的需求讨论记录。

## 使用方式

```
/view-requirements
```

或者直接说"查看需求"

## 功能

1. 查找当前最新的需求文档（`docs/requirements/` 目录下修改时间最近的文件）
2. 读取并展示给用户
3. 如果有多个未完成的需求（讨论进度不是"已完成"），展示列表让用户选择

## 输出格式

展示时使用以下格式：

```markdown
## 需求文档：<主题>

> 创建时间：xxx

### 确认的需求点
<内容>

### 待确认
<内容>

### 讨论进度
<内容>
```

## 注意事项

- 如果没有找到任何需求文档，提示用户"暂无需求讨论记录"
- 只展示"未完成"的需求，已完成的不主动展示
- 用户可以指定查看某个具体文档："查看 2月20日 的需求"
