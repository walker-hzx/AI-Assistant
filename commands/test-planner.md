# 测试设计

使用此命令设计 E2E 测试场景和测试用例。

## 使用方式

```
/test-planner
```

## 功能

1. **需求分析** - 理解功能需求
2. **页面功能梳理** - 梳理页面交互和功能点
3. **测试场景设计** - 设计覆盖核心路径的测试场景
4. **测试用例编写** - 编写具体的测试用例

## 适用场景

- 开始为某个功能编写 E2E 测试前
- 需要梳理复杂业务逻辑时
- 设计测试策略和覆盖范围时

## 输出

- 测试场景列表
- 测试用例设计
- Playwright 测试代码模板
