# 调度优化

## 职责

基于历史执行数据，分析调度效率，优化角色分配策略和时间预估。

**定位**：管家的"学习"环节

---

## 优化层次

### 层次 1：任务级别

每次任务完成后分析：
- 任务分配是否合理？
- 时间预估是否准确？
- 遇到什么问题？

### 层次 2：阶段级别

每个里程碑完成后分析：
- 哪些任务总是一起出现？
- 哪些角色组合效率最高？
- 里程碑设置是否合理？

### 层次 3：系统级别

长期积累后分析：
- 用户偏好什么风格？
- 哪些类型需求增长最快？
- 需要新增/调整角色吗？

---

## 分析维度

### 1. 时间分析

```
预计时间 vs 实际时间：
- [ ] 预估准确（误差 < 20%）
- [ ] 预估过长（实际快很多）
- [ ] 预估过短（实际慢很多）

分析原因：
- 为什么预估不准？
- 哪个环节出了问题？
```

### 2. 角色分析

```
角色使用效率：
- [ ] 需求分析师：使用 N 次，平均耗时 X
- [ ] 计划制定者：使用 N 次，平均耗时 Y
- ...

角色组合效果：
- 组合 A + B：效率高/低
- 组合 X + Y：效率高/低
```

### 3. 问题分析

```
常见问题类型：
- [ ] 需求不清晰 → 需要更早触发 brainstorming
- [ ] 计划遗漏步骤 → 需要更详细的计划
- [ ] 验证不充分 → 需要增加验证环节

反复出现的问题：
- 问题 1：出现 N 次
- 问题 2：出现 M 次
```

## 优化存储

### 存储位置

`docs/plans/coordinator-optimization.md`

### 积累机制

```
每次任务/阶段完成后：
1. 记录执行数据
2. 不立即生成优化建议
3. 积累 5+ 次后生成分析
4. 应用优化建议
```

---

## 自我改进

### 改进触发

- 每完成 5 次任务
- 用户主动触发（输入"优化调度"）
- 发现明显问题模式

### 改进方式

```
根据分析结果：
1. 调整任务类型判断逻辑
2. 优化角色选择策略
3. 改进时间预估模型
4. 建议新增角色/技能
```
