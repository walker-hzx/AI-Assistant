# 任务派发

## 职责

按照 coordinator-planning 制定的方案，调度角色执行，监控进度，收集结果，更新执行日志。

**定位**：管家的"执行"环节

**核心原则**：
- 只负责"执行派发"（按方案调用角色）
- 不负责"决定派发给谁"（那是 coordinator-planning 的职责）

---

## 派发机制

### 调用 Skill 的方式

```
使用 Skill 工具调用：
- 技能名称
- 参数（如需要）

示例：
/brainstorming
/executing-plans
/debugging
```

### 调用 Agent 的方式

```
使用 Task 工具调用：
- subagent_type
- 描述
- prompt

示例：
Task(subagent_type="ai-assistant:planner", prompt="...")
```

---

## 执行流程

### 步骤 1：开始执行

**向用户展示**：
```
## 开始执行

**任务**：[任务名称]
**当前阶段**：里程碑 1 / 总 N 个里程碑
**调度**：[Skill/Agent 名称]
```

### 步骤 2：执行监控

**每个阶段执行中**：
- 等待阶段完成
- 记录执行状态
- 遇到问题及时处理

**问题处理**：
```
发现问题了：
- 问题描述：[是什么]
- 影响范围：[多大]
- 解决方案：[如何处理]

需要调整方案吗？(Y/N)
```

### 步骤 3：结果收集

**每个阶段完成后**：
- 收集产出物
- 验证是否达标
- 记录完成状态

### 步骤 4：阶段衔接

**阶段间检查**：
- 当前阶段是否完成？
- 产出是否满足要求？
- 可以进入下一阶段吗？

---

## 日志更新

### 更新时机

| 时机 | 更新内容 |
|------|---------|
| 阶段开始 | 开始时间、当前状态 |
| 阶段完成 | 完成时间、产出物 |
| 问题发现 | 问题描述、影响 |
| 方案调整 | 调整内容、理由 |

### 日志格式

```markdown
## 执行日志

### 里程碑 1：[名称]

**状态**：进行中 / 已完成 / 有问题

**开始时间**：YYYY-MM-DD HH:MM

**执行记录**：
- HH:MM：开始执行
- HH:MM：完成步骤 1
- HH:MM：完成步骤 2

**产出物**：
- [文件/功能描述]

**遇到的问题**（如有）：
- 问题描述

### 里程碑 2：[名称]

**状态**：待开始 / 进行中
```

---

## 进度汇报

### 向用户汇报的格式

```
## 执行进度

**任务**：[名称]
**当前阶段**：里程碑 X / 总 Y 个
**进度**：XX%

### 已完成
- [x] 里程碑 1：完成
- [x] 里程碑 2：完成

### 进行中
- [ ] 里程碑 3：正在执行...

### 待处理
- [ ] 里程碑 4

**预计剩余时间**：约 Z 分钟
```

---

## 完成处理

### 任务完成时

**收集最终结果**：
- 所有产出物
- 执行摘要
- 遇到的问题及解决

**更新日志为完成状态**：
```markdown
## 执行完成

**完成时间**：YYYY-MM-DD HH:MM
**总耗时**：X 分钟

**最终结果**：
- [产出物列表]

**执行摘要**：
- 总阶段数：N
- 按时完成：X
- 有问题：Y
- 调整次数：Z
```
