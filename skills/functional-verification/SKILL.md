# 功能验证

## 概述

验证功能是否正常工作，符合需求规格。

**核心理念**：证据先于声明。

---

## 验证流程

### 1. 需求对照

- 读取需求文档
- 列出验收标准
- 逐项对照

### 2. 功能测试

| 测试类型 | 说明 |
|----------|------|
| 正常流程 | 核心功能是否正常 |
| 边界条件 | 边界输入是否正确处理 |
| 异常流程 | 错误情况是否正确处理 |

### 3. 证据收集

- 截图
- 日志
- 测试结果

---

## 验证清单

### 核心功能

| 检查项 | 说明 |
|--------|------|
| 功能正常 | 核心功能可运行 |
| 数据正确 | 业务数据处理正确 |
| 状态正确 | 状态变更符合预期 |

### 边界条件

| 检查项 | 说明 |
|--------|------|
| 空值处理 | 空输入是否正确处理 |
| 边界值 | 最大/最小值是否正确 |
| 并发 | 并发场景是否正确 |

### 异常处理

| 检查项 | 说明 |
|--------|------|
| 错误提示 | 错误信息是否友好 |
| 异常恢复 | 异常后是否可恢复 |
| 日志记录 | 错误是否被记录 |

---

## 输出格式

```markdown
## 功能验证

### 验收标准对照

| 标准 | 状态 | 证据 |
|------|------|------|
| 标准1 | ✅/❌ | 截图/日志 |
| 标准2 | ✅/❌ | 截图/日志 |

### 测试结果

- 正常流程：X 通过
- 边界条件：X 通过
- 异常流程：X 通过

### 结论
[通过/不通过]
```

---

## 注意事项

- 每项必须有证据
- 不只测试正常流程
- 边界和异常同样重要
