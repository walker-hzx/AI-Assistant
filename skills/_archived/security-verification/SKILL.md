# 安全验证

## 概述

检查代码是否存在安全漏洞，确保无安全风险。

**核心理念**：安全是底线，不可妥协。

---

## 验证维度

### 1. 输入安全

| 检查项 | 说明 |
|--------|------|
| 参数验证 | 所有输入是否验证 |
| SQL 注入 | 是否使用参数化查询 |
| XSS | 输出是否转义 |

### 2. 认证授权

| 检查项 | 说明 |
|--------|------|
| 身份验证 | 是否验证身份 |
| 权限控制 | 是否有权限检查 |
| 敏感数据 | 是否保护敏感信息 |

### 3. 密钥安全

| 检查项 | 说明 |
|--------|------|
| 硬编码 | 是否有硬编码密钥 |
| 环境变量 | 密钥是否通过环境变量 |
| 日志记录 | 敏感信息是否写入日志 |

---

| A04 | 不安全设计 |

---

## 验证流程

### 1. 代码扫描

- 静态分析代码
- 识别潜在风险
- 检查安全模式

### 2. 工具检查

- 安全扫描工具
- 依赖漏洞检查
- 配置检查

### 3. 人工审查

- 关键逻辑审查
- 认证授权审查
- 数据保护审查
