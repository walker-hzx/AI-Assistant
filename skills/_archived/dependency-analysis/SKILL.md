# 依赖分析

## 概述

分析任务间的依赖关系，识别哪些可以并行、哪些必须串行。

**核心理念**：最大化并行执行，最小化等待时间。

---

## 依赖类型

### 1. 串行依赖

必须先完成 A 才能做 B：
```
任务A → 任务B
```

场景：
- B 依赖 A 的输出
- B 需要 A 创建的资源
- B 修改 A 的文件

### 2. 并行依赖

可以同时执行：
```
任务A
任务B
```

场景：
- 无依赖关系
- 操作不同文件
- 独立的功能模块

### 3. 外部依赖

依赖外部因素：
- 第三方 API
- 环境配置
- 其他人完成的工作

---

## 分析流程

1. **列出所有任务**
   - 获取任务列表

2. **分析每项依赖**
   - 前置任务是什么？
   - 可以并行吗？

3. **构建依赖图**
   - 串行链
   - 可并行组

4. **优化顺序**
   - 最大化并行
   - 最小化等待

---

## 并行优化原则

1. **同批次 ≤ 3 个任务**
   - 太多并行难以管理

2. **先并行无依赖任务**
   - 快速产出

3. **按依赖顺序执行**
   - 串行部分按顺序

---

## 注意事项

- 不要假设任务可以并行
- 检查是否有隐藏依赖
- 考虑资源冲突
