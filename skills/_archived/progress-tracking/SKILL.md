# 进度追踪

## 概述

记录执行状态，更新任务进度，追踪问题。

**核心理念**：记录每个阶段做什么，便于追溯和分析。

---

## 追踪内容

### 1. 任务状态

| 状态 | 说明 |
|------|------|
| 待开始 | 还未执行 |
| 进行中 | 正在执行 |
| 已完成 | 执行成功 |
| 有问题 | 遇到问题 |

### 2. 执行记录

| 内容 | 说明 |
|------|------|
| 开始时间 | 任务开始时间 |
| 结束时间 | 任务结束时间 |
| 耗时 | 总耗时 |
| 问题 | 遇到的问题 |
| 解决方案 | 如何解决 |

### 3. 里程碑状态

| 里程碑 | 状态 | 完成度 |
|--------|------|--------|
| 里程碑1 | 进行中 | 60% |
| 里程碑2 | 待开始 | 0% |

---

## 记录格式

```markdown
## 执行日志

### 任务 1：[名称]
- **状态**：已完成
- **开始**：10:00
- **结束**：10:15
- **耗时**：15分钟
- **问题**：无

### 任务 2：[名称]
- **状态**：进行中
- **开始**：10:20
- **问题**：接口返回格式不对
- **解决**：调整解析逻辑
```

---

## 更新时机

| 时机 | 记录内容 |
|------|----------|
| 任务开始 | 任务名、开始时间 |
| 任务完成 | 结束时间、结果 |
| 遇到问题 | 问题描述、解决方案 |
| 里程碑完成 | 完成状态、交付物 |

---

## 注意事项

- 记录要简洁
- 关键节点必记
- 问题要记录解决
- 便于后续追溯
