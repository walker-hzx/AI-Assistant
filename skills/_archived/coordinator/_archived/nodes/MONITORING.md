# Coordinator 执行监控节点

> 执行过程中的监控相关节点

---

## 任务理解自动化

> **自动分析任务描述，识别任务类型和复杂度**

### 执行时机

- 步骤 0 完成后，步骤 1 之前
- 每次接收新任务时必须执行

### 执行步骤

1. 分析用户输入，提取关键词
2. 根据关键词识别任务类型
3. 计算复杂度评分（1-5）
4. 评估影响范围和可逆性
5. 计算置信度

### 关键词匹配

```python
# 任务类型识别
if (包含["修复","报错","bug","问题","崩溃"]) → bugfix
else if (包含["做","实现","添加","开发","创建"]) → feature
else if (包含["重构","优化","整理"]) → refactor
else if (包含["调研","了解","分析"]) → research
else if (包含["文档","说明"]) → docs
else if (包含["测试","用例"]) → test

# 复杂度调整
if (包含["复杂","大量","系统级"]) → complexity +2
if (包含["简单","小","单个"]) → complexity -1

# 影响范围
if (包含["全局","整个","所有"]) → scope = system
if (包含["这个","当前"]) → scope = single-file
```

### 置信度与后续处理

| 置信度 | 处理方式 |
|--------|---------|
| >= 0.8 | 自动通过，无需确认 |
| 0.5-0.8 | 展示识别结果，用户确认 |
| < 0.5 | 标记为"待定"，询问用户 |

### 输出

- 任务类型
- 复杂度评分
- 影响范围
- 可逆性
- 风险等级
- 置信度

## 执行监控增强

> **不只是事后检查，执行过程中也要监控质量**

### 执行监控流程

#### 1. 初始化监控

- 读取计划文档，提取任务列表和预估时间
- 计算总时间预算 = 所有任务预估时间之和
- 设置预警阈值：时间预算的 80%

#### 2. 执行前检查

- 检查产出物目录是否存在
- 检查依赖文件是否就绪
- 如有问题，提前预警

#### 3. 执行中监控（每个里程碑完成后）

- 进度检查：已完成任务数 / 总任务数
- 时间检查：已用时间 / 预算时间
- 产出检查：实际产出 vs 预期产出
- 质量检查：代码是否符合规范

#### 4. 预警处理

| 预警级别 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 绿色 | 正常 | 继续执行 |
| 黄色 | 进度滞后 > 20% | 记录警告，继续执行 |
| 橙色 | 产出不符合预期 | 暂停，检查原因 |
| 红色 | 严重偏离计划 | 停止，报告用户 |

#### 5. 里程碑检查点

每个里程碑完成后执行：
- [ ] 产出文件存在
- [ ] 代码无明显语法错误
- [ ] 测试可运行
- [ ] 进度在预算内

---

## 监控指标

| 指标 | 说明 | 阈值 |
|------|------|------|
| 进度 | 已完成任务/总任务数 | < 50% 时关注 |
| 时间 | 已用时间/预算时间 | > 80% 时预警 |
| 产出 | 实际产出 vs 预期产出 | 不匹配时预警 |
| 质量 | 代码规范/测试覆盖 | 不达标时预警 |
