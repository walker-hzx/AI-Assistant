# Coordinator 质量评估节点

> 步骤6-7相关的质量评估节点

---

## 代码审查

> **审查代码质量，确保符合规范**

**【强制】审查结果必须写入文档，且由 Subagent 创建**

### 执行

1. 审查代码质量（规范、安全、性能、可维护性）
2. 使用 Task 调用 ai-assistant:code-reviewer 创建审查文档：`docs/reviews/<task>-review.md`

### 审查报告必须包含

- 审查项
- 审查结果（通过/失败）
- 问题及修复建议

### 门控检查

- 审查报告存在 → 读取审查结果
- 审查报告不存在 → 停止并提示用户
- 审查通过 → 进入下一步
- 审查未通过 → 修复后重新审查（最多 3 次）

---

## 安全审查触发

> **敏感操作必须经过安全审查**

**【强制】涉及敏感操作的任务必须完成安全审查**

### 触发条件 - 必须执行安全审查

- 涉及用户认证/授权（登录、注册、权限）
- 涉及支付/资金
- 涉及敏感数据（身份证、银行卡、手机号等）
- 涉及权限变更
- 涉及API密钥、密码、token等
- 涉及数据导出/导入

### 跳过条件

- 任务不涉及上述敏感操作

### 执行

1. 判断是否涉及敏感操作
2. 如果是，使用 Task 调用 ai-assistant:security-reviewer 创建安全审查文档
3. 文档路径：`docs/reviews/<task>-security.md`

### 门控检查

- 敏感操作 + 无安全审查 → 【阻断】必须完成安全审查
- 非敏感操作 → 跳过
- 安全审查通过 → 继续

---

## 审查结果质疑

> **验证审查报告的完整性**

1. 使用 Task 调用 ai-assistant:skeptics 进行审查结果质疑：
   - 任务描述：对审查报告进行质疑
   - 质疑类型：review
   - 被质疑文档：docs/reviews/<task>-review.md
   - 输出：docs/skeptics/<task>-review-skeptics.md
2. 读取质疑报告
3. 根据严重程度处理

---

## 质量评估

> **评估各阶段产出物质量**

**【强制】评估结果必须写入文档，且由 Subagent 创建**

**此步骤仅复杂任务需要**。

### 执行

1. 使用 Task 调用 ai-assistant:evaluator 评估产出物质量
2. 读取评估结果
3. 根据结果判断：
   - 合格 → 进入完成
   - 不合格 → 退回重做
4. 生成汇总报告：`docs/quality/<task>-summary.md`

### 质量评估报告必须包含

- 评估维度
- 评估结果（合格/不合格）
- 问题及优化建议

### 门控检查

- 汇总报告存在 → 进入完成
- 汇总报告不存在 → 停止并提示用户

---

## quietMode/unattendedMode 处理

| 节点 | quietMode | unattendedMode |
|------|-----------|----------------|
| 代码审查 | 执行 | 执行 |
| 安全审查触发 | 跳过 | 跳过 |
| 审查结果质疑 | 跳过 | 跳过 |
| 质量评估 | 跳过 | 跳过 |
