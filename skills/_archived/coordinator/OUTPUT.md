# 文档输出规范

> 只在 M/L 档任务需要时创建文档

---

## 文档位置

| 文档类型 | 路径 | 何时创建 |
|----------|------|---------|
| 计划文档 | `docs/plans/<task>-plan.md` | L 档任务 |
| 进度文档 | `docs/progress/<task>-progress.md` | 跨会话任务 |
| 验证报告 | `docs/verification/<task>-verification.md` | L 档任务（可选） |
| 审查报告 | `docs/reviews/<task>-review.md` | L 档任务（可选） |

---

## 计划文档模板

```markdown
# 计划：{任务名称}

## 需求理解
- 核心目标：{一句话}
- 验收标准：
  - [ ] 标准 1
  - [ ] 标准 2

## 实施步骤

### 里程碑 1：{名称}
- [ ] 任务 1（executor）
- [ ] 任务 2（executor）
- 验证：{如何验证这个里程碑}

### 里程碑 2：{名称}
- [ ] 任务 3（executor）
- 验证：{如何验证}

## 风险点
- {风险及应对}
```

---

## 进度文档模板（跨会话用）

```markdown
# 进度：{任务名称}

## 当前状态
- 状态：{进行中/暂停/待确认}
- 当前阶段：{里程碑 N}
- 下一步：{下次启动时做什么}

## 已完成
- [x] {已完成项}

## 待完成
- [ ] {待完成项}

## 注意事项
- {需要记住的上下文}
```
