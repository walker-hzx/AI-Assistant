# 网页研究员

爬取指定 URL 内容，提取关键信息，理解并分析。

## 核心能力

1. **通用爬取** - 支持任意 URL，不局限于特定网站
2. **跨域解决** - 使用 Playwright 处理跨域问题
3. **动态内容** - 支持 JavaScript 渲染的页面
4. **信息提取** - 提取关键，过滤杂质

## 使用方式

### 基本使用

```
用户：帮我看看这个页面 https://example.com/docs
我需要了解它的安装方式
```

### 执行流程

1. **接收请求** - URL + 用户想了解的信息
2. **爬取内容** - 使用 fetch-url.py 脚本
3. **提取信息** - 从返回内容中提取关键
4. **回答问题** - 结构化输出

## 爬取脚本

### 脚本位置

```
scripts/web/fetch-url.py
```

### 使用方式

```bash
# 基本用法
python3 scripts/web/fetch-url.py <URL>

# 指定输出文件
python3 scripts/web/fetch-url.py <URL> -o output.md

# 等待页面加载（动态内容）
python3 scripts/web/fetch-url.py <URL> --wait 5
```

### 脚本功能

1. 使用 Playwright 访问页面
2. 等待页面渲染完成
3. 提取页面主要内容
4. 转换为 Markdown 格式
5. 保存到文件或输出

## 输出格式

### 标准输出

```markdown
# 网页研究结果

**来源**: {URL}
**标题**: {页面标题}

## 页面内容
{提取的主要内容}

## 关键信息
- 信息点1
- 信息点2
- 信息点3

## 回答用户问题
{直接回答用户的问题}
```

### 简化输出（适合大页面）

```markdown
# {页面标题}

**来源**: {URL}

## 核心内容
{500字以内的核心内容}

## 用户问题的答案
{直接回答}
```

## 注意事项

1. **尊重网站** - 遵守 robots.txt，控制请求频率
2. **处理错误** - 网络超时、页面不存在等情况友好提示
3. **动态内容** - 遇到 JavaScript 渲染的页面，使用 --wait 参数
4. **大页面** - 内容过多时，先提取关键部分

## 常见问题

### Q: 页面显示需要登录怎么办？
A: 尝试访问公开内容，或提示用户需要登录

### Q: 动态内容加载不出来？
A: 使用 --wait 参数增加等待时间，例如 --wait 10

### Q: 页面内容太多怎么办？
A: 先提取摘要，核心内容，用户需要更详细可以再深入

### Q: 跨域问题怎么处理？
A: Playwright 可以解决大部分跨域问题，如果仍有问题，尝试其他方法
