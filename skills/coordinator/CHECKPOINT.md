# 检查点定义

> 本文档定义任务执行过程中的检查点。根据任务类型选择需要的检查点。

---

## 检查点列表

### 检查点 0：任务状态检测

**检查内容**：
- [ ] 扫描 docs/coordinator/ 目录
- [ ] 判断是否有进行中的任务

---

### 检查点 1：调度记录创建

**检查内容**：
- [ ] `docs/coordinator/<task>-coordinator.md` 已创建
- [ ] 任务基本信息已填写
- [ ] 状态为"进行中"

---

### 检查点 2：需求分析（可选）

**检查内容**：
- [ ] `docs/intent/<task>-intent.md` 已创建（如果需要）
- [ ] 用户真实意图已明确
- [ ] 任务类型已判断

**此步骤可选**：简单任务可能不需要

---

### 检查点 3：制定计划（可选）

**检查内容**：
- [ ] `docs/plans/<task>-plan.md` 已创建（如果需要）
- [ ] 角色组合已确定
- [ ] 每个任务有执行者声明

**此步骤可选**：简单任务可能不需要

---

### 检查点 4：执行轮次完成

**检查内容**（每轮执行后）：
- [ ] 执行日志 `docs/execution/<task>-execution-N.md` 已创建
- [ ] Subagent 输出文档存在
- [ ] 任务状态已更新

---

### 检查点 5：功能验证（可选）

**检查内容**：
- [ ] `docs/verification/<task>-verification.md` 已创建（如果需要）
- [ ] 功能验证通过

**此步骤可选**：调研任务等不需要

---

### 检查点 6：代码审查（可选）

**检查内容**：
- [ ] `docs/reviews/<task>-review.md` 已创建（如果需要）
- [ ] 代码质量审查通过

**此步骤可选**：调研任务等不需要

---

### 检查点 7：质量评估（仅复杂任务）

**检查内容**：
- [ ] `docs/quality/<task>-summary.md` 已创建（仅复杂任务）
- [ ] 质量评估通过

**此步骤仅复杂任务需要**

---

### 检查点 8：任务完成

**检查内容**：
- [ ] 调度记录状态为"已完成"
- [ ] 所有必要步骤已完成

---

## 检查函数

### 通用检查函数

```
【检查文档是否存在】
使用 Glob 工具检查：
- docs/coordinator/<task>-coordinator.md
- docs/intent/<task>-intent.md
- docs/plans/<task>-plan.md
- docs/execution/<task>-execution-N.md
- docs/verification/<task>-verification.md
- docs/reviews/<task>-review.md

【检查必填字段】
使用 Read 工具读取文档内容，检查必填字段是否存在
```

---

## 失败处理原则

| 失败类型 | 处理方式 |
|----------|----------|
| 文档缺失 | 创建后重试 |
| 字段缺失 | 补充后重试 |
| 验证失败 | 修复后重试（最多 3 次） |
| 审查失败 | 修复后重试或接受风险 |
| 达到上限 | 通知用户，人工介入 |
