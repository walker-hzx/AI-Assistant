# 智能助手

**一句话需求，智能调度执行**

---

## 使用方式

```
/assistant [你的需求]

例如：
/assistant 帮我加个用户登录功能
/assistant 修复登录页面的拼写错误
/assistant 帮我优化下接口性能
```

---

## 工作流程

当你输入需求后，管家会按以下流程处理：

### 1. 接收需求【提示用户】

```
【管家】收到需求：帮我加个用户管理功能
```

### 2. 意图分析【提示用户】

```
【管家】正在分析意图...
    ↓
【管家】判断为：实现类 - 普通复杂度
```

### 3. 方案制定【提示用户】

```
【管家】正在制定方案...
    ↓
【管家】方案已制定，准备派发任务
```

### 4. 任务派发【提示用户】

```
【管家】派发给 brainstorming（需求分析）
【管家】派发给 writing-plans（制定计划）
【管家】派发给 executing-plans（执行计划）
```

### 5. 执行完成【提示用户】

```
【管家】任务已完成
【管家】执行摘要：...
```

---

### 完整流程示例

```
【管家】收到需求：帮我加个用户管理功能

【管家】正在分析意图...
    → 用户想要新增一个用户管理模块
    → 涉及前后端开发
    → 判断为：实现类 - 普通复杂度

【管家】正在制定方案...
    → 需要：需求分析 → 计划制定 → 代码实现 → 验证
    → 预计 3 个阶段

【管家】派发给 brainstorming（需求分析）
    → 等待分析完成...

[... 后续阶段类似提示 ...]

【管家】任务已完成
【管家】执行摘要：用户管理功能已开发完成，包含增删改查
```

---

## 典型场景

| 场景 | 输入 | 管家会 |
|------|------|--------|
| 新功能 | `/assistant 加个用户管理功能` | 需求分析 → 计划 → 执行 → 验证 |
| Bug修复 | `/assistant 登录接口报错了` | 调试 → 修复 → 验证 |
| 代码优化 | `/assistant 优化下用户查询性能` | 分析 → 优化 → 验证 |
| 简单任务 | `/assistant 改个变量名` | 直接处理，记录结果 |

---

## 底层实现

此入口会调用 `coordinator` Agent 来处理任务。

**关键**：管家在每个阶段都会输出状态提示，让你知道当前在哪一步。

具体流程由 coordinator 的子技能完成：
- coordinator-intent：意图分析 → 输出状态提示
- coordinator-planning：方案制定 → 输出状态提示
- coordinator-dispatch：任务派发 → 输出派发提示
- coordinator-optimization：调度优化 → 记录分析数据
