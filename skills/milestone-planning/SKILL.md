# 里程碑划分

## 概述

将任务划分为多个里程碑，每个里程碑有明确的检查点和交付物。

**核心理念**：阶段性交付，持续验证。

---

## 划分原则

### 里程碑特征

| 特征 | 说明 |
|------|------|
| 可独立运行 | 里程碑内任务闭环 |
| 有明确交付物 | 完成后能看到效果 |
| 有检查点 | 可以暂停确认 |

### 阶段划分维度

| 维度 | 说明 |
|------|------|
| 依赖关系 | 核心功能 → 增强功能 |
| 价值交付 | 先完成可独立使用的 |
| 风险 | 低风险 → 高风险 |

---

## 划分流程

1. **分析任务列表**
   - 了解所有任务

2. **识别可独立交付**
   - 哪些可以先完成

3. **按维度划分阶段**
   - 依赖/价值/风险

4. **定义里程碑**
   - 名称
   - 交付物
   - 检查点

---

## 输出位置

**保存路径**：`docs/plans/<task>-plan.md`

---

## 输出格式

```markdown
## 里程碑划分

### 里程碑 1：[名称]
**目标**：xxx
**交付物**：xxx

**任务**：
- [ ] 任务A
- [ ] 任务B

**检查点**：
- [ ] 功能可运行
- [ ] 核心测试通过

---

### 里程碑 2：[名称]
...
```

---

## 里程碑数量

| 复杂度 | 里程碑数 |
|--------|----------|
| 简单 | 1-2 个 |
| 中等 | 2-3 个 |
| 复杂 | 3-5 个 |

---

## 注意事项

- 每个里程碑应该可独立验证
- 里程碑间有明确依赖
- 不要把太多任务放在一个里程碑
