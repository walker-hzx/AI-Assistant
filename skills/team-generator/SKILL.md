# Team Generator

## 概述

动态生成 Claude Code Agent Team，让多个角色并行协作工作。

**核心原则：** 按需生成，简洁高效，标准化输出。

---

## 团队配置结构

### 目录结构

```
teams/
├── [团队名称]/
│   └── config.json
```

### 配置格式

```json
{
  "name": "团队名称",
  "description": "团队描述",
  "workflow": {
    "顺序": "工作流程顺序",
    "沟通原则": "沟通基本原则"
  },
  "members": [
    {
      "role": "角色",
      "name": "成员名称",
      "职责": "职责描述",
      "skills": ["使用的技能"],
      "沟通": {
        "方式": "沟通方式",
        "频率": "沟通频率",
        "内容": ["沟通内容"],
        "触发条件": ["什么情况需要沟通"],
        "找谁": {
          "问题类型": "找谁沟通"
        }
      },
      "prompt": "完整的 prompt"
    }
  ]
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| name | 团队名称 |
| description | 团队描述 |
| workflow | 工作流程（顺序 + 沟通原则） |
| members | 成员列表 |
| members[].role | 角色名称 |
| members[].name | 成员名称 |
| members[].职责 | 职责描述 |
| members[].skills | 使用的技能列表 |
| members[].沟通 | 沟通配合规范 |
| members[].prompt | 完整的 prompt |
| members[].initialization | 角色初始化配置（可选） |

### 初始化配置

每个角色可以配置 `initialization` 字段，用于团队激活后 Lead 给角色分配初始化任务：

```json
{
  "initialization": {
    "identity": "你是 XXX 角色的身份描述",
    "responsibilities": ["职责1", "职责2"],
    "skills": ["使用的技能"],
    "communication": {
      "找谁": {"问题类型": "角色名"},
      "reportTo": "汇报对象"
    },
    "notes": "补充说明"
  }
}
```

### 沟通规范

| 字段 | 说明 | 示例 |
|------|------|------|
| 方式 | 沟通方式 | 直接沟通、任务列表分配 |
| 频率 | 沟通频率 | 每完成1个任务、每天、遇到问题时 |
| 内容 | 沟通内容 | 进度、问题、结果 |
| 触发条件 | 什么情况需要沟通 | 任务完成、遇到问题 |
| 找谁 | 什么问题找谁 | 技术问题找Backend |

## 创建团队流程

### 从配置加载

1. 读取 `teams/[团队名称]/config.json`
2. 解析配置
3. 生成自然语言描述
4. 创建团队

### 生成自然语言描述

基于配置自动生成：

```
请创建一个 [团队名称] 来 [任务]。

## 团队结构
- [角色1]：[职责]（使用 skill: [技能]）
- [角色2]：[职责]（使用 skill: [技能]）
...

## 沟通规范
- 沟通方式：[方式]
- 沟通频率：[频率]
- 触发条件：[触发条件]
- [问题类型]找[角色]

## 工作流程
[workflow.顺序]

## 关键要求
- [沟通原则]
- [其他要求]

请使用 Sonnet 模型，每个成员独立工作。
```

## 常见问题

### Q1: 如何选择团队模板？

| 判断标准 | 推荐团队 |
|----------|---------|
| 不了解项目现状 | 代码分析团队 |
| 需求不明确 | 需求评审团队 |
| 需要开发新功能 | 开发团队 |
| 有代码需要审查 | 代码审查团队 |
| 遇到问题需要排查 | 调试团队 |

### Q2: 如何自定义团队？

在 `teams/` 目录下创建新的配置文件：
```
teams/
└── my-team/
    └── config.json
```

按配置格式填写即可。

### Q3: 团队成员数量多少合适？

- **最佳**：3-5 人
- **最少**：2 人（Lead + 1 队友）
- **最多**：不超过 7 人

```
请根据 [团队名称] 配置生成团队创建描述
```

**生成模板**：

```
请创建一个 [团队名称] 来 [任务]。

## 团队结构
- [角色1]：[职责]（使用 skill: [技能]）
- [角色2]：[职责]（使用 skill: [技能]）
...

## 沟通规范
- 沟通方式：[方式]
- 沟通频率：[频率]
- 触发条件：[触发条件]
- [问题类型]找[角色]
```

### 功能 2: 生成 Lead 初始化任务描述

根据团队配置的 `initialization` 字段，生成 Lead 给各角色分配初始化任务的描述。

**使用方式**：告诉用户可以用以下格式生成描述

```
请根据 [团队名称] 配置生成 Lead 初始化任务描述
```

**生成模板**：

```
## 团队初始化任务

请按照以下配置初始化各角色：

### [角色1名称]
- **身份**: [identity]
- **职责**:
  - [responsibility1]
  - [responsibility2]
- **可用技能**: [skills]
- **沟通方式**: [communication.找谁]
- **备注**: [notes]

### [角色2名称]
...

## 初始化要求
1. 依次向各角色传达初始化配置
2. 确认各角色理解自己的职责
3. 明确沟通汇报关系
4. 准备好后开始团队工作
```

**注意**：如果配置中没有 `initialization` 字段，则无法生成此描述。
