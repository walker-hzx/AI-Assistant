# 接口验证

## 概述

验证 API 接口是否正确，返回数据是否符合预期。

**核心理念**：确保前后端接口正确对接。

---

## 验证流程

### 1. 接口检查

- 检查 API 端点
- 检查请求方法
- 检查请求参数

### 2. 响应验证

| 检查项 | 说明 |
|--------|------|
| 状态码 | 2xx 成功，4xx/5xx 错误 |
| 响应体 | 数据结构是否符合预期 |
| 错误处理 | 错误响应是否正确 |

### 3. 数据验证

- 字段类型
- 字段值
- 必填字段

---

## 接口测试

### 正常请求

```bash
curl -X GET /api/users
```

预期：
- 状态码 200
- 返回用户列表

### 异常请求

```bash
curl -X GET /api/users/999
```

预期：
- 状态码 404
- 返回错误信息

---

## 验证清单

### 请求层面

| 检查项 | 说明 |
|--------|------|
| URL 正确 | 端点路径正确 |
| 方法正确 | GET/POST/PUT/DELETE |
| 参数正确 | 必填参数完整 |

### 响应层面

| 检查项 | 说明 |
|--------|------|
| 状态码 | 正确区分成功/失败 |
| 响应体 | 数据结构正确 |
| 错误信息 | 错误描述清晰 |

### 数据层面

| 检查项 | 说明 |
|--------|------|
| 字段完整 | 必填字段不缺失 |
| 类型正确 | 字段类型匹配 |
| 值有效 | 值在合理范围内 |
